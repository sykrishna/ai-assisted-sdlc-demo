# Documentation Hub

## Purpose

The `docs/` directory is the primary source of truth for product and technical intent. It is organized to support spec-driven delivery from discovery to implementation.

## Subdirectories

- `product-specs/`: business goals, user requirements, and acceptance criteria.
- `architecture/`: system design, technical constraints, and decision rationale.
- `api-contracts/`: interface definitions and contract governance.
- `ai-engineering-rules/`: AI-assisted engineering standards and controls.
- `observability/`: foundational standards for logging, tracing, health, and monitoring.

## Usage Expectations

- Documents should be versioned and reviewed through pull requests.
- Significant changes should include rationale, impact, and reviewer sign-off.
- Future implementation work should cite source documents for traceability.
